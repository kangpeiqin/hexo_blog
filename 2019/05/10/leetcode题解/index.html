为什么要做题呢？其实做题只是为了让你保持一种编程的状态，保持一种每天解决问题的习 惯，因为编程其实就是解决实际问题的过程啦。每天一点点，其实不会花费太多的时间啊，以下我将分类记下一些leetcode的题解。因为有很多这样的博客了，而且也分类写的很好了，我想做的只是记录一下我的历程啦，在其中也参照了很多他人的博客，分类是参考某博客的分类。

LinkedList

常用的解题方法

  • 遍历链表
  • 利用递归的思想，如反转链表
  • 双指针：快慢指针

Remove Duplicates from Sorted List

思路：遍历链表，在遍历时判断当前结点的值与后面结点的值是否相等，时间复杂度O(n)。

Remove Duplicates from Sorted List II

思路：采用双指针，一个指针指向需要保留的结点，一个指针对有重复值的结点进行查找。

Linked List Cycle

思路：采用快慢指针的思想。如果存在环，快指针终归会追上慢指针

String

常用的解题方法

  • 运用哈希
  • 使用双指针

Implement strStr()

思想：1.暴力破解，如果发生不匹配主串可能需要回溯。2.KMP算法。

Valid Anagram

思想：创建一个哈希数组，如果两个字符串含有同样的单词则将该位置的值置为0。

Valid Palindrome

思路：采用双指针的思想，在这当中，使用了java提供的一些函数。

Array

常用的解题方法

  • 遍历数组，标记变量
  • 双指针

Remove Duplicates from Sorted Array

思路：使用双指针，一个指针为遍历指针；另一个指针指向结果数组的每个位置，判断该位置要填充的元素。

Remove Duplicates from Sorted Array II

思路：同上题的思路相同，只是多一个变量count对元素的数量进行统计。

Remove Element

思路：双指针，在遍历时用另一个指针确定结果数组的每一个位置的最终值

stack

需要掌握的思想

  • 模拟进出栈
  • 用栈解决问题的思想

Min Stack

思路：用两个栈，一个栈为最小值栈，一个栈为真实数据栈。或者用一个栈也可以。
